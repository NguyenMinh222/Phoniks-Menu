# PHONIKS QR Menu — production Netlify build

## Что внутри

- `index.html` — структура страницы.
- `style.css` — стабильный мобильный дизайн без тяжёлых анимаций.
- `script.js` — логика языков, поиска, категорий, lightbox и PWA.
- `menu-data.js` — все блюда и категории.
- `service-worker.js` — production cache: основные файлы всегда сначала из сети, картинки кэшируются мягко.
- `_headers` — правила кэша для Netlify.
- `_redirects` — fallback для Netlify.

## Важные изменения

1. Версия сделана на базе стабильной логики, а не на неудачной v4.
2. Поиск больше не ищет по названию категории, поэтому не подтягивает лишние блюда.
3. Поиск работает по словам: все введённые слова должны быть в названии/описании/ID блюда.
4. Блюдо может иметь несколько категорий:

```js
category: "soups",
categories: ["soups", "vegan"]
```

5. Добавлены категории:

```js
vegan
sets
```

6. При смене категории меню возвращается вверх.
7. Язык и категория больше не сохраняются навсегда в `localStorage`, поэтому при новом открытии меню стартует с главной страницы и русского языка.
8. Для Netlify добавлены `_headers` и улучшенный `service-worker.js`, чтобы новые деплои не ломались из-за старого кэша.

## Как заменить фото

Клади изображения в папку `images` и прописывай в `menu-data.js`:

```js
image: "images/pho-bo.webp"
```

Лучший формат для QR-меню: `.webp`, ширина 1200–1600 px, вес до 300–600 KB.

## Если меняешь картинку с тем же именем

Netlify и браузер могут держать фото в кэше до нескольких часов. Для моментального обновления лучше менять имя файла:

```js
image: "images/pho-bo-v2.webp"
```


## Правка размера карточек

Карточки возвращены к нормальному размеру: убран `content-visibility`, который мог создавать завышенную высоту карточек и дергания, а изображения ограничены по высоте через `max-height`.
